# 安全说明

GhostTap 是一个强大的自动化工具，我们非常重视安全和隐私。

## 🔒 设计安全原则

### 1. 最小权限原则

- **无障碍权限**：仅用于读取界面和执行点击/输入操作
- **悬浮窗权限**：仅用于显示任务状态和控制按钮
- **网络权限**：仅用于连接用户指定的服务器

### 2. 敏感操作保护

**自动暂停场景**：
- 检测到"支付"、"付款"等关键词
- 检测到"密码"、"安全验证"等关键词
- 涉及金额输入或确认

**暂停后流程**：
1. 悬浮窗显示暂停状态
2. 用户手动完成敏感操作
3. 用户点击"继续"恢复 AI 控制

### 3. 数据保护

| 数据类型 | 处理方式 |
|---------|---------|
| UI 树数据 | 仅实时传输，不持久化存储 |
| 用户凭证 | 仅存储 user_id，密码等敏感信息永不存储 |
| 通信数据 | TLS 1.2/1.3 加密传输 |
| 日志数据 | 本地存储，不上传云端 |

### 4. 开源可审计

- 服务端代码完全开源
- 用户可以自建服务器
- 所有通信协议公开透明

## ⚠️ 使用警告

### 合法使用

- ✅ 个人设备自动化
- ✅ 自动化测试
- ✅ 辅助操作
- ✅ 学习和研究

### 禁止行为

- ❌ 控制他人设备
- ❌ 恶意自动化（刷量、作弊等）
- ❌ 违反平台服务条款的操作
- ❌ 任何非法用途

## 🔐 自部署安全建议

### 服务端

1. **使用 HTTPS/WSS**
   ```
   必须配置 TLS 证书
   禁止使用明文 WebSocket (ws://)
   ```

2. **访问控制**
   ```
   配置防火墙，限制访问 IP
   定期更换 user_id（后续支持）
   ```

3. **日志清理**
   ```
   UI 数据不记录或定期清理
   敏感操作额外标记
   ```

### 客户端

1. **权限管理**
   - 无障碍权限：按需开启，用完可关闭
   - 悬浮窗权限：仅在需要时显示

2. **网络环境**
   - 避免使用公共 WiFi
   - 使用可信的自建服务器
